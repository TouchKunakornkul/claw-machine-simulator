📑 GAME DESIGN DOCUMENT (GDD)Project Working Title: Claw Machine Tycoon & Simulator (ชื่อชั่วคราว)Genre: Physics-Based Simulator / Shop Management (Tycoon)Platform: PC (Windows)Target Engine: Unity (C#)Graphics Style: Low-Poly Stylized หรือ Semi-Realistic (เน้นแสงเงาสวยในพื้นที่จำกัด)1. Executive Summary (ภาพรวมของเกม)เกมจำลองการเป็นพ่อค้าตู้คีบตุ๊กตาที่ผู้เล่นต้องใช้ฝีมือสู้กับระบบฟิสิกส์ตู้คีบที่สมจริง (และกลไกแอบโกงเหมือนชีวิตจริง) คีบของรางวัลจากตู้สาธารณะเพื่อนำมาสะสม จัดแสดง หรือตั้งราคาขายในร้านค้าของตัวเอง เพื่อทำกำไรและขยายกิจการไปสู่ตู้คีบระดับพรีเมียมที่ยากและแพงขึ้น2. Core Gameplay Loop (วงจรการเล่นหลัก)[ ทำงานพาร์ทไทม์ / เก็บเหรียญ ] ➔ [ หยอดเหรียญคีบตุ๊กตา (Physics-Based) ] ➔ [ นำของรางวัลมาตั้งขาย / จัดโชว์ ] ➔ [ ได้กำไร / ขยายร้าน ] ➔ [ ปลดล็อกตู้ระดับสูงขึ้น ]
3. Key Features (ระบบเด่นของเกม)Realistic Claw Machine Mechanical Physics: จำลองระบบขาคีบ 3 จังหวะ (หนีบแน่นตอนล่าง ➔ ปล่อยหลวมตอนยก ➔ เหวี่ยงตอนเลื่อน) พร้อมค่าความหยุ่น (Soft-body) และแรงเสียดทาน (Friction) ของของรางวัลแต่ละประเภทProgression Tier: ระบบไต่ระดับความยากและมูลค่าของรางวัล (ตู้โชห่วย ➔ ตู้ห้างสรรพสินค้า ➔ ตู้อากิฮาบาระ)Shop Display & Management: ระบบเปิดท้ายขายของและจัดหน้าร้าน ตั้งราคาขายตามเทรนด์ความต้องการของ NPCSafety Net System: ระบบทำงานพาร์ทไทม์เช็ดตู้เกมกวาดพื้น (ได้เงินน้อยแต่ได้ชัวร์) สำหรับประทังชีวิตเมื่อเงินหมดตัว4. Technical Specifications (การเซ็ตติ้งใน Unity)Physics Step: ปรับ Fixed Timestep ใน Unity ให้ต่ำลง (เช่น 0.01) เพื่อการคำนวณการชน (Collision) ที่แม่นยำ ขาคีบไม่ทะลุวัตถุJoints: ใช้ Configurable Joint หรือ Character Joint สำหรับจำลองสายเคเบิลขาคีบที่หย่อนและเหวี่ยงได้ตามแรงเฉื่อยColliders: วัตถุของรางวัลใช้ Convex Mesh Collider ที่ละเอียดเพื่อให้เกิดการ "เกี่ยว" กันตามหลักฟิสิกส์จริงๆ🗺️ DEVELOPMENT ROADMAP (แผนการพัฒนา)แผนงานนี้แบ่งออกเป็น 4 ระยะ (Phases) เพื่อให้คุณค่อยๆ พัฒนาจากฟิสิกส์แกนหลักไปจนถึงระบบจัดการร้านค้า[ Phase 1: Core Physics ] ➔ [ Phase 2: Gameplay Loop ] ➔ [ Phase 3: Tycoon Systems ] ➔ [ Phase 4: Polish & Launch ]
🟩 Phase 1: Core Physics Prototype (สร้างระบบฟิสิกส์แกนหลัก)เป้าหมาย: ทำตู้คีบจำลองที่ขยับและหนีบวัตถุได้ในฉากทดสอบ (Greybox)สิ่งที่ต้องทำ:สร้างโมเดลตู้คีบและกล่อง/ลูกบอลทดลองอย่างง่ายเขียน C# Script ควบคุมขาคีบ (ขยับแกน X/Z, กดปุ่มทิ้งตัวแกน Y)เขียนระบบหนีบวัตถุ (ขากรรไกรหุบเข้าหากันเมื่อชนพื้นหรือกดซ้ำ)ทดลองปรับค่าแรงหนีบ (Force) และแรงเสียดทาน (Friction)🟨 Phase 2: Gameplay Progression & UI (สร้างวงจรเกมและเมนู)เป้าหมาย: มีระบบเงิน คีบแล้วได้ของจริง และย้ายตู้เล่นได้สิ่งที่ต้องทำ:สร้างระบบตัวแปรเก็บเงิน (Wallet) และระบบหักเงินต่อการกด 1 ครั้งสร้าง Trigger Zone ที่ก้นถังปล่อยของ เมื่อวัตถุตกใส่ จะทำการทำลายวัตถุ (Destroy) แล้วเพิ่มของชิ้นนั้นเข้าสู่คลัง (Inventory) ของผู้เล่นทำตู้คีบ 2 ระดับ: ตู้คีบง่าย (ลูกบอล) และตู้คีบยาก (กล่องสี่เหลี่ยมผิวลื่น)ทำระบบ "ทำงานพาร์ทไทม์" (กดปุ่มค้างไว้ 3 วินาที เพื่อรับเงินประทังชีวิตทีละ 2 เหรียญ)🟧 Phase 3: Tycoon & Economy (ระบบร้านค้าและการจัดการ)เป้าหมาย: นำของที่คีบได้มาสร้างรายได้เพื่อขยายตู้สิ่งที่ต้องทำ:สร้างฉาก "ร้านค้าของผู้เล่น" ที่มีโต๊ะวางขายของเขียนระบบ NPC เดินสุ่มเข้ามาในร้าน (AI NavMesh แบบง่าย) และเดินมาตรวจดูสินค้าที่โต๊ะทำระบบตั้งราคาขาย (ถ้าตั้งราคาเหมาะสม NPC จะหยิบของและเงินในกระเป๋าเราจะเพิ่มขึ้น)ทำประตูหรือเมนูสำหรับจ่ายเงินปลดล็อกแผนที่/ตู้คีบระดับสูง🟦 Phase 4: Polish, Audio & Launch (เก็บงาน แสงสีเสียง และปล่อยเกม)เป้าหมาย: เพิ่มความน่าดึงดูดและเตรียมวางขายบน Steamสิ่งที่ต้องทำ:ใส่เสียงเอฟเฟกต์ (เสียงกลไกขยับ, เสียงเหรียญหยอด, เสียงดนตรีตู้เกม)ใส่ระบบแสงไฟ LED กะพริบในตู้คีบ และระบบสั่น (Haptic Feedback)ขัดเกลากราฟิก ใส่โมเดลตุ๊กตาผ้าและกล่องฟิกเกอร์ที่สวยงามจริงBuild เกมส่งออกเป็นไฟล์ .exe สำหรับ Windows และทดสอบบั๊กฟิสิกส์
