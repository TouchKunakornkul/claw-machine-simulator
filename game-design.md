📑 GAME DESIGN DOCUMENT (GDD) - Shared World EditionProject Working Title: Claw Machine World: Market TycoonGenre: Physics-Based Simulator / Shared-World EconomyPlatform: PC (Windows / Steam)Target Engine: Unity (C#)Server Architecture: Asynchronous Central Database (Web API เช่น PlayFab หรือ Firebase ยืนยันข้อมูลผ่าน Steam)1. Executive Summary (ภาพรวมของเกม)เกมจำลองตู้คีบตุ๊กตาฟิสิกส์สมจริงในโลกออนไลน์ที่แชร์ข้อมูลร่วมกัน (Shared World) ผู้เล่นจากทั่วโลกจะเข้ามาร้านอาเขตเดียวกัน เพื่อต่อคิวและแย่งชิงของรางวัลที่มี "จำนวนจำกัดจริงๆ บนเซิร์ฟเวอร์" ของรางวัลที่คีบได้จะกลายเป็นสินทรัพย์ในคลังของผู้เล่น (Steam Inventory) ซึ่งสามารถนำไปจัดแสดงในร้านของตัวเอง หรือนำไปตั้งราคาขายเปลี่ยนเป็นเงินจริงบน Steam Marketplace ได้2. Core Gameplay Loop (วงจรการเล่นหลัก)[ Daily Login / เติมเงินเพื่อรับเหรียญ ] ➔ [ ต่อคิวเข้าตู้คีบ (1 คนต่อ 1 ตู้) ] ➔ [ คีบของรางวัล Limited (ตัดยอดจาก Server) ] ➔ [ ไอเทมเข้า Steam Inventory ] ➔ [ นำไปจัดโชว์ในร้านค้าตัวเอง หรือ ตั้งขายบน Steam Market ]
3. Key Systems & Features (ระบบเด่นของเกม)A. One Player Per Machine (ระบบ 1 ตู้เล่นได้ทีละคน)Queue System: หากตู้คีบนั้นมีผู้เล่นอื่นกำลังเล่นอยู่ ผู้เล่นคนอื่นจะเห็นสถานะ "Busy" และสามารถกด "ต่อคิว" ได้ โดยจะเห็นหน้าจอไลฟ์สตรีม (สตรีมเฉพาะค่าพิกัดวัตถุ) ของคนที่กำลังเล่นอยู่สดๆ เพิ่มความกดดันและน่าติดตามTime Limit: ผู้เล่นที่ได้สิทธิ์ครองตู้จะมีเวลาจำกัด (เช่น ตาละ 45 วินาที หรือเล่นติดต่อกันได้ไม่เกิน 5 ครั้ง) เพื่อป้องกันการยึดตู้แช่B. Limited Scarcity & Server Drop (ของรางวัลจำนวนจำกัด)Global Stock: เซิร์ฟเวอร์ส่วนกลางจะทำหน้าที่ควบคุมจำนวนของรางวัล เช่น "สัปดาห์นี้มี ฟิกเกอร์มังกรทอง ปล่อยออกมาแค่ 100 ตัวทั่วโลก" ยิ่งของเหลือน้อย ค่าความ FOMO จะยิ่งสูงServer Dynamic Fill: เมื่อผู้เล่นคีบของออกไปได้ เซิร์ฟเวอร์จะบันทึกพิกัดวัตถุที่เหลือในตู้นั้นใหม่ เพื่อให้ผู้เล่นคนต่อไปเจอสภาพตู้ที่เปลี่ยนไปตามฟิสิกส์จริงที่คนก่อนหน้าทำไว้C. Game Economy & Monetization (ระบบการเงินและการเติมเงิน)Token (เหรียญหยอดตู้): เป็นค่าเงินหลักในการเล่น ไม่สามารถฟาร์มได้แบบไร้ขีดจำกัดสายฟรี: ได้จากระบบ Daily Login หรือการทำภารกิจจำลอง เช่น เช็ดตู้เกมกวาดพื้น (จำกัดจำนวนครั้งต่อวัน)สายเปย์: เติมเงินจริงซื้อแพ็กเกจเหรียญเพื่อสิทธิ์ในการหยอดตู้ระดับพรีเมียมSteam Market Integration: ทางผู้พัฒนาจะได้ส่วนแบ่งเปอร์เซ็นต์ (Transaction Fee) ทุกครั้งที่มีการซื้อขายตุ๊กตาหรือของรางวัลระหว่างผู้เล่นบน Steam Market4. Technical Specifications & Server LogicClient-Side Physics: เพื่อป้องกันเน็ตแลค การคำนวณฟิสิกส์ขาคีบและตุ๊กตาทั้งหมดจะรันบนคอมพิวเตอร์ของผู้เล่น (Client) 100%Server-Side Verification: เมื่อตุ๊กตาตกท่อสำเร็จ Client จะส่งสัญญาณไปที่ Server เพื่อทำการ Verify & Mint ไอเทมเข้า Steam Inventory ของผู้เล่นตัวจริง เพื่อป้องกันการแฮกจำลองฟิสิกส์🗺️ REVISED ROADMAP (แผนการพัฒนาปรับปรุงใหม่)[ Phase 1: Core Physics ] ➔ [ Phase 2: Server & Login ] ➔ [ Phase 3: Steam & Queue ] ➔ [ Phase 4: Bot & Soft Launch ]
🟩 Phase 1: Core Physics & UI Prototype (1-2 เดือน)ทำระบบขยับขาคีบ 3 จังหวะ และตั้งค่าฟิสิกส์วัตถุใน Unity ให้เสถียรบน Windows PCทำระบบ UI ปุ่มกดหยอดเหรียญและหักคะแนนเหรียญในเครื่องตัวเองก่อน🟨 Phase 2: Database & Daily Login (1 เดือน)เชื่อมต่อ Unity เข้ากับเซิร์ฟเวอร์ฐานข้อมูล (เช่น PlayFab ซึ่งฟรีและใช้ง่าย)ทำระบบสมัครสมาชิก/ล็อกอิน และระบบ Daily Login (เช็กเวลาเซิร์ฟเวอร์เพื่อแจกเหรียญฟรีทุกๆ 24 ชั่วโมง)ทำระบบตัดยอดของรางวัลบนฐานข้อมูลเมื่อคีบตกท่อ🟧 Phase 3: Steam Integration & Queue Logic (1-2 เดือน)เชื่อมต่อ Steamworks API เข้ากับ Unity เพื่อดึงระบบ ID ของ Steam และเปิดระบบ Steam Inventoryเขียนสคริปต์ระบบต่อคิวหน้าตู้คีบ (Queue Sync) และการแชร์สภาพตู้ (Object Coordinate Sync) หลังจากจบตาการเล่น🟦 Phase 4: Simulated Bots & Soft Launch (1 เดือน)เขียนระบบ AI Bots ให้สุ่มตัดยอดของรางวัลและขยับตู้คีบเสมือนในช่วงที่คนเล่นยังน้อย เพื่อไม่ให้ร้านดูเงียบเหงาเปิดหน้า Steam Store สะสมยอด Wishlist และเปิดทดสอบช่วง Closed Beta เพื่อดูความเสถียรของเซิร์ฟเวอร์
